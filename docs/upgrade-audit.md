# 추천 서비스 업그레이드 점검 결과

점검일: 2026년 6월 15일

## 결론

서비스는 포용금융에 **기여할 수 있는 추천·접근성 도구**다. 그러나 모든 추천을 포용금융이라고 부르는 것은 과장이다.

직접적인 포용금융 효과는 소상공인과 취약계층·농어촌 주민에게 자격상품을 누락 없이 연결하고 금융비용을 비교할 때 발생한다. 일반 고객에게는 연령이나 인기순으로 유리한 상품이 가려지지 않게 하는 금융소비자 보호·정보 접근성 개선 성격이 더 강하다.

금융위원회는 포용금융 목표로 소상공인·취약계층의 금융비용과 채무부담 완화, 금융소외계층의 금융 접근성 제고를 제시한다. 이 기준에 맞춰 본 서비스는 `자격상품 노출`, `실현 가능 이자`, `쉬운 이용`을 별도로 관리해야 한다.

## 발견한 문제와 수정

### 기간별 금리 오류

기존 버전은 12개월 표시금리를 6·24·36개월에도 재사용했다. 공식 상세 금리 API의 기간별 만기이자지급식 기본금리로 변경했다.

### 금리 최신성

2026년 6월 11일 이후 공식 금리가 0.15%p 변경됐지만 로컬 데이터는 이를 알리지 못했다. 2026년 6월 12일 기준 금리로 갱신하고 4일 이상 지난 정보에는 재확인 경고를 표시한다.

### 우대상한 설명

조건 합계가 상품 우대상한을 넘을 때 모든 조건이 금리에 더해지는 것처럼 보일 수 있었다. 실제 반영금리와 남은 우대폭을 분리했다.

### 추천 점수 과잉 개입

초기 업그레이드에서는 조건 간편성이 기본 금리 우선순위를 뒤집었다. 기본 목표는 세전 예상이자를 절대 우선하고, 편의·포용 기준은 사용자가 해당 목표를 선택한 경우에만 순위를 바꾸도록 수정했다.

### 지원대상 오표시

포용금융 배지는 상품명이나 연령만으로 표시하지 않는다. 소상공인 또는 취약계층·농어촌 등 입력 자격과 상품 지원대상이 실제로 맞을 때만 표시한다. 고령층 편의상품은 핵심 취약계층 상품과 구분해 보조 수준으로 평가한다.

## UI·접근성 점검

- 핵심 입력과 세부 우대조건을 분리해 첫 화면의 인지 부담 축소
- 예시 고객 버튼으로 입력 부담 감소
- 모든 주요 버튼 최소 높이 44px 이상
- 주요 문자색 대비 6.31:1 이상
- 4px 고대비 키보드 포커스 제공
- 본문 바로가기 링크 제공
- 큰 글씨·고대비 전환 제공
- 결과와 상담 답변을 상태영역으로 안내
- 390px 화면에서 가로 스크롤 없음
- 자유입력 대신 선택형 상담 질문 6개 제공

WCAG 2.2 AA는 일반 텍스트 명도 대비 4.5:1 이상과 최소 24×24 CSS 픽셀 수준의 조작영역 또는 충분한 간격을 요구한다. 본 프로토타입은 주요 조작영역을 44px 수준으로 넉넉하게 잡았다.

## 선택형 RAG 평가

상품 도우미는 외부 생성형 모델을 호출하지 않는다.

1. 현재 고객 조건으로 추천 결과를 다시 계산한다.
2. 선택한 질문에 따라 금리, 우대조건, 편의기능, 가입대상 문서를 검색한다.
3. 검색된 상품정보만 정해진 설명 템플릿에 넣는다.
4. 공식 상품상세 링크를 근거로 제공한다.

자유입력형 챗봇보다 기능은 좁지만 금융상품 조건의 환각 위험, 개인정보 전송, 어려운 질문 작성 부담을 줄인다.

## 남은 한계

- 일반과세 15.4%만 가정하며 비과세종합저축 여부는 계산하지 않는다.
- 중도해지이자는 실제 해지시점별로 계산하지 않고 주의문만 제공한다.
- 고객의 실제 우대조건 충족 여부는 자기선택 값이며 은행 원장 API로 검증하지 않는다.
- 상품설명 변경을 완전히 자동 반영하지 않으므로 운영에서는 일별 동기화가 필요하다.
- 합성 고객 6명 평가는 실제 전환율이나 포용금융 성과를 증명하지 않는다.

## 운영 성과지표

- 가입 가능한 전체 상품의 추천 후보 포함률 99% 이상
- 예상금리와 가입 확정금리 오차 0.05%p 이하
- 소상공인·취약계층 자격 고객의 전용상품 노출률
- 일반 고객과 지원대상 고객 간 추천 실패율 차이
- 큰 글씨·고대비·선택형 상담 사용률
- 상품 상세조회율, 가입완료율, 중도해지율, 관련 민원율
