# 우체국 예금 맞춤 추천 서비스

연령대별 인기상품을 단순 노출하는 대신, 가입 가능한 우체국 정기예금을 모두 비교해 고객이 **실제로 받을 수 있는 금리, 예상 세후이자, 편의기능, 포용금융 지원 적합성**을 설명하는 프로토타입입니다.

> 금리 기준일: 2026-06-12 · 데이터 검증일: 2026-06-15

## 문제와 해결 방향

기존 연령 중심 추천은 같은 연령대라도 가입 조건과 거래 실적이 다른 고객에게 더 유리한 상품을 누락할 수 있습니다.

이 프로젝트는 다음 원칙으로 추천을 개선합니다.

1. 가입할 수 없는 상품만 먼저 제외합니다.
2. 선택 기간에 맞는 공식 기본금리를 적용합니다.
3. 고객이 충족한 우대조건과 상품별 우대상한을 계산합니다.
4. 예상 세전·세후이자와 편의기능을 함께 보여줍니다.
5. 연령은 상품을 숨기는 기준이 아니라 실제 우대조건과 동률 보조기준으로만 사용합니다.
6. 소상공인·취약계층 등 자격이 확인된 경우에만 포용금융 적합성을 표시합니다.

## 주요 기능

- 6·12·24·36개월별 공식 기본금리 적용
- 가입금액·기간·채널·전용자격 검증
- 상품 우대상한을 반영한 실현 가능 금리 계산
- 일반과세 15.4% 기준 예상 세후이자 표시
- 최고이자·조건 간편성·자금 유동성·포용금융 목적별 추천
- 추천 이유, 적용된 우대조건, 추가 확인조건 설명
- 자유입력 없이 버튼만 선택하는 로컬 RAG 상품 도우미
- 큰 글씨·고대비·키보드 포커스·모바일 반응형 UI
- 금리 기준일과 데이터 신선도 경고

## 빠른 실행

Python 3만 필요하며 외부 패키지는 사용하지 않습니다.

```bash
python3 app.py
```

브라우저에서 [http://127.0.0.1:8000](http://127.0.0.1:8000)에 접속합니다.

8000 포트가 사용 중이면 다른 포트를 지정할 수 있습니다.

```bash
python3 app.py --port 8001
```

명령행 추천도 사용할 수 있습니다.

```bash
python3 recommend.py
python3 evaluate.py
```

## 추천 알고리즘

기본 목표인 `highest_interest`는 실현 가능한 예상이자를 절대 우선합니다. 다른 추천 목표는 사용자가 직접 선택했을 때만 조건 간편성, 유동성, 포용금융 적합성에 가중치를 부여합니다.

| 추천 목표 | 설명 |
| --- | --- |
| `highest_interest` | 실현 가능한 세전이자가 큰 상품 우선 |
| `easy_conditions` | 우대조건을 충족하기 쉬운 상품 우선 |
| `liquidity` | 분할해지·비상금 출금 등 자금 활용이 편한 상품 우선 |
| `inclusive` | 확인된 지원 자격에 맞는 포용금융 상품 우선 |

상품별 결과에는 기본금리, 반영 우대금리, 적용 조건, 추가 확인조건, 세전·세후 예상이자와 공식 상품정보 링크가 포함됩니다.

## 선택형 RAG 도우미

`src/advisor.py`는 현재 추천 결과와 `data/products.json`의 구조화된 상품 문서를 검색해 정해진 답변 템플릿에 근거를 채웁니다.

- 1위 추천 이유
- 1위와 2위 비교
- 추가 우대조건
- 유동성 편의
- 포용금융 자격상품
- 창구 상담 필요 여부

사용자는 질문을 직접 입력하지 않고 버튼만 선택합니다. 고객 입력은 외부 LLM이나 외부 API로 전송하지 않으며, 상품조건을 임의 생성할 위험을 줄이기 위해 답변 범위를 의도적으로 제한했습니다.

## API

| 메서드 | 경로 | 역할 |
| --- | --- | --- |
| `GET` | `/api/health` | 서버 상태 확인 |
| `POST` | `/api/recommend` | 고객 조건 기반 상품 추천 |
| `GET` | `/api/advice/questions` | 선택 가능한 도우미 질문 조회 |
| `POST` | `/api/advice` | 현재 추천 결과에 근거한 답변 생성 |
| `GET` | `/api/evaluation` | 합성 고객 시나리오 평가 결과 |

상세 요청·응답 예시는 [API 연동 문서](docs/api-integration.md)를 참고하세요.

## 포용금융 판단

서비스 전체를 곧바로 포용금융 상품이라고 단정하지 않습니다. 대신 다음 두 영역에서 기여 가능성을 구분합니다.

- **직접 지원:** 소상공인정기예금과 이웃사랑정기예금처럼 지원대상이 명시된 상품을 자격 고객에게 누락 없이 연결
- **접근성 지원:** 고령층과 디지털 취약고객도 큰 글씨, 고대비, 쉬운 설명, 선택형 상담으로 상품을 비교하도록 지원

실제 운영 단계에서는 자격 고객 전용상품 노출률, 확정금리 오차, 가입완료율, 민원율, 중도해지율을 함께 측정해야 포용금융 효과를 검증할 수 있습니다. 자세한 평가는 [업그레이드 점검 보고서](docs/upgrade-audit.md)에 정리했습니다.

## 검증

```bash
python3 -m unittest discover -s tests -v
python3 scripts/check_official_rates.py
python3 evaluate.py
node --check static/app.js
```

현재 검증 결과:

- 자동 테스트 11개 통과
- 공식 정기예금 12개 최고금리 확인
- 공식 상세 API 기간별 기본금리 29건 일치
- 390px 모바일 화면에서 가로 넘침 없음
- 주요 문자색 명도 대비 6.31:1 이상
- 주요 버튼·체크 항목 터치 영역 44px 이상
- 브라우저 오류·경고 없음

6개 합성 고객 시나리오에서 기존 연령 중심 기준선의 최적상품 Top-1 적중률은 `33.3%`, 고객 1명당 평균 예상 세전이자 개선은 `60,833원`이었습니다. 이는 실제 고객 성과 추정치가 아니라 기능 비교용 시뮬레이션입니다.

```text

├── app.py                         # 로컬 HTTP·JSON API 서버

├── evaluate.py                    # 기준선 비교 평가
├── data/
│   └── products.json              # 상품·금리·우대규칙·포용금융 태그
├── src/
│   ├── recommender.py             # 자격 검증과 추천 계산

│   └── evaluation.py              # 합성 고객 시나리오 평가
├── static/                        # 접근성 반응형 웹 UI
├── scripts/
│   └── check_official_rates.py    # 공식 목록·상세 금리 검증
├── tests/
│   └── test_recommender.py        # 추천·자격·RAG 단위 테스트
└── docs/
    ├── api-integration.md
    ├── proposal.md
    └── upgrade-audit.md
```
